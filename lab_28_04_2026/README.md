# ЛР 28/04/2026 — Разработка алгоритма для реализации метода RAG

Реализация находится в файле `lab_28_04_2026/rag_pipeline.py`.

## Что делает скрипт

1. Загружает `graph.json` и строит текстовые абзацы по узлам онтологии:
   - название узла;
   - атрибуты (`params_values`);
   - ближайшие входящие/исходящие связи.
2. Вычисляет эмбеддинги узлов через функции из `lab_10_03_2026/embeddings.py`.
3. Первая фаза retrieval:
   - эмбеддинг вопроса;
   - поиск `N` наиболее релевантных узлов;
   - генерация предварительного ответа через text-generation модель Qwen.
4. Вторая фаза retrieval:
   - эмбеддинг предварительного ответа;
   - поиск `M` дополнительных узлов;
   - объединение `N + M` и генерация финального ответа.

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Пример запуска (только retrieval, без генерации LLM)

```bash
python -m lab_28_04_2026.rag_pipeline --question "Над какими проектами работал Серый А.С.?" --graph graph.json --no-llm
```

## Пример запуска с LLM Qwen2.5 7B Instruct

```bash
python -m lab_28_04_2026.rag_pipeline --question "Над какими проектами работал Серый А.С.?" --graph graph.json --llm-model Qwen/Qwen2.5-7B-Instruct
```

По умолчанию в скрипте используется модель `Qwen/Qwen2.5-7B-Instruct`.

## Параметры

- `--n-first` — число узлов в первой выборке (по умолчанию `5`).
- `--m-second` — число узлов во второй выборке (по умолчанию `4`).
- `--embedding-model` — модель эмбеддингов (по умолчанию мультиязычная).
- `--llm-model` — text-generation модель (`deepseek-ai/...`).
- `--no-llm` — отключить генерацию и проверить только поиск.
